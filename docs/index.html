Tya

Small language, plain tools

CoffeeScript feel, Go-style practicality.

Tya is a small indentation-based dynamic language for writing readable scripts and turning them into simple binaries with a cross-build-friendly workflow.

Run Tya Read the guide Reference

The current implementation is written in Go and emits C; the language is moving toward practical single-binary distribution and easy cross-builds.

hello.tya 
greet = user -> "Hello, {user["name"]}"

user =
  name: "komagata"
  age: 20

if user["age"] >= 20
  print greet user

Language

Designed to be small enough to understand.

Tya uses indentation for blocks, simple values, first-class functions, object-style dictionaries, error values, and a compact standard library.

Indentation-based

Blocks use two-space indentation with explicit lexer support for INDENT and DEDENT.

Dynamic values

Use nil, booleans, numbers, strings, arrays, dictionaries, functions, and error values.

Expression-friendly

String interpolation, implicit returns, multiple returns, and try keep small programs direct.

Checked before running

The checker catches undefined variables, const reassignment, duplicate declarations, and placement errors.

Runs two ways

Run through the interpreter, or emit C on the path toward single-binary builds and simple cross-platform distribution.

Self-hosting work

Tya-written compiler components exist under selfhost/; full language parity is still in progress.

Try it

Run examples from the repository.

The command-line tool can print tokens, check unused names, interpret a file, emit C, or compile and run through C.

  1. Run a source filego run ./cmd/tya examples/hello.tya
  2. Compile and execute through Cgo run ./cmd/tya run examples/hello.tya
  3. Inspect generated Cgo run ./cmd/tya --emit-c examples/arithmetic.tya

Self-hosting

The compiler is being rewritten in Tya.

The repository contains Tya-written lexer, parser, checker, and C code generator sources. The supported-subset bootstrap gate is complete; full language parity is still tracked in the roadmap.

The next major work is moving the self-host parser away from line-oriented node strings and toward structured AST parsing.

  • selfhost/lexer.tya, selfhost/parser.tya, selfhost/checker.tya, and selfhost/codegen_c.tya exist.
  • sh scripts/selfhost_bootstrap_check.sh is the main bootstrap gate.
  • Full completion requires removing generated-tool fallback behavior and matching the Go implementation for the full language.
Tya is a small language implementation project built for clarity, experiments, and self-hosting work.
