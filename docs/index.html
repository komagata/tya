Guide

Tya Guide

Tya is a small indentation-based dynamic language inspired by CoffeeScript. This guide is for reading from top to bottom.

Run a Program

go run ./cmd/tya examples/hello.tya
go run ./cmd/tya run examples/hello.tya

tya run compiles the source to C, builds it with cc, and runs the resulting binary.

Names

Use snake_case for variables, functions, modules, and object properties. Use SCREAMING_SNAKE_CASE for constants.

Conditions

  print "adult"

Loops

  print count
  count = count + 1
items = ["a", "b"]

  print item

for item, index in items
  print "{index}: {item}"

Use of to iterate dictionary keys and values.

user = { name: "komagata", age: 20 }

for key, value of user
  print "{key}: {value}"

Functions

Functions use ->. The last expression is returned implicitly.

greet = name -> "Hello, {name}"

print greet "Tya"
double = value ->
  result = value * 2
  result

Use return when returning early or returning multiple values.

parse_user = text ->
  if text == ""
    return nil, error "empty user"
  return { name: text }, nil

Arrays and Dictionaries

items = [1, 2]
push items, 3
print items[0]
counter =
  count: 0

  inc: ->
    @count = @count + 1
    @count

print counter.inc()

Errors

Tya uses error values, not exceptions. Inside a function, try propagates the error part of a value, err result.

user, err = parse_user ""
if err
  print err.message
load_user = text ->
  user = try parse_user(text)
  user["name"]

Modules

Import another .tya file from the same directory. Each module exposes exactly one public top-level binding, and that binding must match the file name.

import greeting

print greeting.hello("komagata")
# greeting.tya
greeting =
  hello: name -> "Hello, {name}"

Reference

Tya Reference

This is the compact language reference for Tya. It favors exactness over tutorial flow.

Source Files

Tya source files use the .tya extension. A program is a list of statements. Newlines separate statements.

Truthiness

nil and false are falsey. Other values are truthy.

Lexical Structure

Comments start with # and continue to the end of the line. Blocks are indentation-based. One indentation level is two spaces.

# comment
name = "Tya"

Values

Runtime values are nil, boolean, integer, float, string, array, dictionary, function, and error.

Strings support escapes for \", \\, \n, and \t. Interpolation uses {expression} inside a string.

Functions and Errors

A function creates a child scope. A block function returns the last expression unless return is used. A function can return multiple values.

error "message" creates an error value with a message property. try expression is valid inside a function. If the error part is truthy, the current function returns nil, err.

Modules and Execution

import module_name loads module_name.tya from the same directory as the importing file. Each module file exposes exactly one public top-level binding.

go run ./cmd/tya --tokens file.tya
go run ./cmd/tya --check-unused file.tya
go run ./cmd/tya --emit-c file.tya
go run ./cmd/tya run file.tya

The runner and C emitter load stdlib/prelude.tya.

API

Tya API

Built-in functions available to Tya programs.

Core

print, panic, exit

print writes a value and a newline. panic stops with an error. exit terminates with a status code.

error

error returns an error value with a message property.

panic "bad state"
exit 1

err = error "file not found"
print err.message
