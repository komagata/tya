# Tya Strict Semantics v1 Audit

Status: current v1.0.0-blocking audit for the dynamic semantics promised by
`docs/SPEC.md`. Tya remains dynamically typed: parser and checker diagnostics
cover source validity that is visible before execution, while runtime errors
cover values whose kinds are only known at execution time.

Diagnostic entries use stable `TYA-E....` codes where the parser or checker
already emits them. Runtime entries name the stable error text used by the
interpreter or C runtime.
